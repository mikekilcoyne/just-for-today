# Friction And Simplification Log

Date: March 20, 2026
Target prototype date: Friday, March 27, 2026

## Current friction

- Too many buttons.
- Not enough clear context.
- Titles breaking or getting smushed.
- App overload.
- Restore / recovery has felt unreliable.
- Some controls create anxiety instead of clarity.
- If the user has to rebuild, recreate, or re-dictate context, the system is already too complicated.
- If the system gets in the way of doing the work, the system is not working.

## Product truth to preserve

- Brain dump is only Stage 1.
- Context should evolve with the day.
- If the user moves to another project or idea, the system should flow with them.
- The app should help reconnect the user with joy in work and flow.
- The north star is Zach's phrase: `Zero Friction`.

## What the next version should optimize for

- Fewer visible controls by default.
- Clearer “what matters right now” context.
- Dynamic context that updates as the day changes.
- Smooth transitions between projects / meetings / follow-ups.
- One working day-to-day prototype that can actually be used daily for 30 days.
- Minimal re-entry of information.
- A system that supports the work instead of demanding work about the work.

## Simplification direction

- Keep the core:
  - brain dump
  - current context
  - next actions
  - meeting follow-up
  - end-of-day handoff
- Collapse or defer everything else until the core feels calm and dependable.
- Default to one primary thread of work at a time, with the ability to switch context gracefully.
- Let context be dynamic instead of static:
  - morning context
  - current working context
  - latest meeting / project context
  - wrap-up context for tomorrow

## Working thesis for rebuild

The app should behave more like a living focus companion than a page of sections.

Possible model:
- `Stage 1`: Brain dump what is in your head.
- `Stage 2`: The app identifies current context and proposes the next meaningful move.
- `Stage 3`: As the day changes, context updates instead of forcing a full restart.
- `Stage 4`: End-of-day wrap-up captures what matters for tomorrow with minimal effort.

## Definition of success by Friday, March 27, 2026

- Working prototype usable on a normal day.
- Clear enough to use without explaining the system to yourself.
- Stable enough to test for 30 consecutive days.
- Strong enough to reduce friction and increase flow.
- Good enough that context survives meetings, follow-ups, and project switching without manual rebuilding.

## Immediate next-step recommendation

Reduce the app to the smallest useful loop:

1. Capture context.
2. Surface the current focus.
3. Support follow-up from meetings and messages.
4. End the day cleanly.
5. Carry tomorrow forward automatically.
